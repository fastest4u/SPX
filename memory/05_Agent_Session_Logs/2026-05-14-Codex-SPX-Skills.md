- Use repo-local Codex Agent Skills rather than inventing unsupported custom CLI slash command files.

## Insights / Learnings

- Codex project workflows should be represented as `.agents/skills` when the goal is reusable, repo-local behavior.
- For this repo, `$spx-*` skills are the closest supported equivalent to custom slash commands and are more portable across Codex CLI, IDE, and app surfaces than a UI-specific command registry.
